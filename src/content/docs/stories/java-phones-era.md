# Java Phones Era

Before app stores standardized everything, mobile software lived in a fragmented world of handset quirks, carrier rules, and tight memory limits.

That fragmentation is part of the point:

- different vendors exposed different capabilities
- distribution workflows were inconsistent
- performance constraints shaped design decisions
- small implementation details changed whether a game felt usable

Stories belong here when they explain why a platform behaved the way it did, not just what buttons to click.
